# 飞达智能 HR 系统 — 开发手册

> 面向二次开发者的架构与扩展指南（v1.2.0）

## 1. 整体架构

飞达是 **单体全栈应用**：后端以 `server/standalone.ts` 为入口，使用 Express 4 风格的 `Router` 挂载于 `/api`；前端是 React 19 SPA（Vite 构建），生产模式下由后端同端口静态托管。

```
请求 → standalone.ts (Express)
        ├─ loginSecurityMiddleware     # 登录速率限制+失败锁定
        ├─ /api/*      → apiAuthMiddleware → apiRouter()
        ├─ /uploads/*  → express.static(uploadDir)
        └─ /*          → 静态托管 dist/client + SPA fallback
```

### 端口设计

| 模式 | 命令 | 前端 | 后端 |
|------|------|------|------|
| 开发 | `npm run dev` | http://localhost:8080 | → vite proxy → :3000 |
| 生产 | `npm start` | http://localhost:3000 | (一体化) |

- `SERVER_PORT` 默认 3000，`SERVER_HOST` 默认 0.0.0.0

## 3. 数据库

- **开发/测试**: SQLite (better-sqlite3)，文件 `data/ehr.db`
- **生产**: PostgreSQL（一行配置切换 `DATABASE_URL=postgres://...`）
- **Agent 会话**: SQLite，文件 `agent/conversations.db`
- **密码存储**: PBKDF2-SHA512（100000轮），兼容旧版简单哈希自动升级

## 4. 模块系统

参照 Odoo `__manifest__.py` 设计。每个模块包含：

```
modules/<name>/
  manifest.json     # 名称/版本/依赖/数据/钩子
  data/*.sql        # 种子数据
  hooks/*.js        # 生命周期钩子
  security/         # 访问控制
```

### manifest.json 示例

```json
{
  "name": "HR核心",
  "version": "1.0.0",
  "depends": ["base"],
  "data": ["data/hr_seed.sql"],
  "hooks": { "post_init": "hooks/post_init.js" }
}
```

### CLI

```bash
npx feida create-module "考勤管理" "HR/考勤"
npx feida list-modules
```

## 5. 安全架构

| 层级 | 组件 | 说明 |
|------|------|------|
| 传输 | HTTPS | 生产环境强制 |
| 认证 | PBKDF2-SHA512 | 100000轮 + 随机盐 |
| API | JWT Token | Bearer auth / dev模式无Token兼容 |
| 登录 | loginSecurityMiddleware | 5次失败锁定30分 / IP速率限制 |
| 权限 | RBAC + 行级 | 权限点 + department/employee过滤 |
| 数据 | company_id | 多公司行隔离 |

## 6. AI Agent 架构

```
用户消息 → AIAssistantPage (React)
         → agentStream() → SSE → POST /api/ai/code-agent/stream
                              → Agent.chat() → runAgentLoop()
                              → LLM Caller → chatCompletionDirect()
                              → 工具执行 → sql_query/bash/grep/read_file...
                              → SSE事件流 → start/step/tool_start/tool_end/done
```

## 7. 开发命令

```bash

npm start            # 启动生产服务 (端口3000)
npm run build:server # 仅构建后端
npm run build:client # 仅构建前端
```
