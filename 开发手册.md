# 飞达系统 — 开发手册

> 面向二次开发者的架构与扩展指南（v1.2.0）
> 新增: ORM引擎 / 低代码平台 / 应用管理 / 模块应用系统

## 1. 整体架构

飞达是 **单体全栈应用**：后端 `server/standalone.ts` 为入口，Express 4 Router 挂载于 `/api`；前端 React 19 SPA（Vite 构建），生产模式同端口静态托管。

```
请求 → standalone.ts (Express)
        ├─ loginSecurityMiddleware     # 登录速率限制+失败锁定
        ├─ /api/*      → apiAuthMiddleware → apiRouter()
        ├─ /uploads/*  → express.static(uploadDir)
        ├─ /addons/*   → express.static(addonsDir)   # app.json等配置
        └─ /*          → 静态托管 dist/client + SPA fallback
```

### 端口

| 模式 | 前端 | 后端 |
|------|------|------|
| 生产 `npm start` | http://localhost:3000 | 一体化 |

`SERVER_PORT` 默认 3000，`SERVER_HOST` 默认 0.0.0.0。

## 3. 飞达 ORM 引擎

自研 Odoo 风格 ORM，支持 Recordset 链式调用。

### 核心类

| 类 | 文件 | 说明 |
|----|------|------|
| `Environment` | `server/modules/orm/environment.ts` | ORM 上下文，Proxy 自动创建 Recordset |
| `Recordset` | `server/modules/orm/recordset.ts` | 惰性记录集，链式操作 |
| `ModelClass` | `server/modules/orm/model-class.ts` | 模型定义（_name/_fields/_hooks） |
| `ModelRegistry` | `server/modules/orm/model-class.ts` | 模型注册表，自动建表 |

### 用法示例

```js
const env = (global).__feida_env;

// CRUD
env.demo_city.create({ name: '北京', code: '010' });
env.demo_city.search({ name: '北京' });
env.demo_city.browse('id123').write({ code: '011' });
env.demo_city.browse('id123').unlink();

// 聚合
env.demo_contact.count();
env.demo_contact.sum('credit');       // 150000
env.demo_contact.avg('credit');       // 75000
env.demo_contact.groupBy('type');     // [{ key:'client', count:1 }]

// 链式
env.demo_contact.search().sorted('credit', true).mapped('name');  // → ['李四','张三']

// 关联
env.demo_contact.search({ email: 'a@test.com' }).resolve('city_id');  // → 城市记录
env.demo_order.browse(id).children('demo_order_line', 'order_id');     // → 明细行

// 公式
_fields: { total: { type: 'formula', formula: 'qty * price' } }
env.test_calc.create({ qty: 10, price: 50 });  // total = 500
```

### 模型定义

```js
// addons/demo_erp/models/demo_contact.js
exports.model = {
  _name: 'demo_contact',
  _description: '联系人',
  _fields: {
    name:    { type: 'char', label: '姓名', required: true },
    city_id: { type: 'many2one', label: '城市', relation: 'demo_city' },
    type:    { type: 'selection', label: '类型', selection: [{label:'客户',value:'client'}] },
    credit:  { type: 'float', label: '信用额度', default: 0 },
  },
  _hooks: {
    beforeWrite: [(id, vals, env) => { if (vals.credit < 0) throw new Error('不能为负'); }],
    afterCreate: [(id, vals, env) => { console.log('新联系人:', vals.name); }],
  },
};
```

### 字段类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `char` | 字符串 | `{ type:'char', label:'名称', required:true }` |
| `text` | 长文本 | `{ type:'text', label:'描述' }` |
| `integer` | 整数 | `{ type:'integer', label:'数量' }` |
| `float` | 小数 | `{ type:'float', label:'金额' }` |
| `boolean` | 布尔 | `{ type:'boolean', label:'启用', default:true }` |
| `date/datetime` | 日期 | `{ type:'date', label:'截止日期' }` |
| `selection` | 下拉 | `{ type:'selection', selection:[{label:'高',value:'high'}] }` |
| `many2one` | 外键关联 | `{ type:'many2one', label:'城市', relation:'demo_city' }` |
| `formula` | 公式计算 | `{ type:'formula', formula:'qty * price' }` |

## 4. 低代码平台

入口: 左侧导航 → **🚀 低代码平台** (`/lowcode`)

### 4步向导

| 步骤 | 内容 |
|------|------|
| Step1 | 应用信息（名称/图标/模块名） |
| Step2 | **模板设计器** — 40+字段类型画廊 + 拖拽画布 + 属性面板 |
| Step3 | 应用预览（导航菜单/文件结构） |
| Step4 | 一键发布（生成模型文件 + 建表 + 注册ORM + 保存app.json） |

### 发布后自动生成

```
addons/<moduleName>/
  manifest.json      # 模块清单
  app.json           # 应用配置（导航/图标/状态）
  models/
    <table>.js       # 模型定义文件
```

### 后端 API

| 接口 | 说明 |
|------|------|
| `POST /api/lowcode/create-module` | 创建模块（生成文件+注册ORM） |
| `POST /api/lowcode/add-field` | 动态添加字段（ALTER TABLE） |
| `POST /api/lowcode/save-app-config` | 保存 app.json |
| `POST /api/lowcode/deploy` | 热加载部署 |
| `GET /api/lowcode/list-modules` | 列出所有 addon 模块 |

## 5. 应用管理

入口: 左侧导航 → **📱 应用管理** (`/apps-manager`)

### 功能

- 卡片式应用列表（图标/名称/版本/状态/模型数/菜单数）
- 统计栏（全部/已发布/草稿/模型总数）
- 状态过滤 + 搜索
- 操作：进入应用 / 发布 / 下架 / 删除

### 应用生命周期

```
新建 → 草稿(draft) → 已发布(published)
         ↑                │
         └─── 下架 ───────┘
                            │
                       归档(archived) → 删除
```

### API

| 接口 | 说明 |
|------|------|
| `GET /api/apps/list` | 应用列表（状态/版本/模型数/菜单数） |
| `POST /api/apps/publish` | 发布/下架/更新版本 |
| `DELETE /api/apps/unlink/:module` | 删除应用（清理文件+ORM） |

## 7. Odoo 模块导入

```bash
# 1. 解析 Odoo Python 模型
python tools/odoo-parser.py "path/to/odoo/models/" > tmp.json

# 2. 生成飞达 JS 模型 + 合并继承
node tools/odoo2feida.js tmp.json addons/<module> <module>

# 3. 重启服务，自动发现
npm start
```
