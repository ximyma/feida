# 00 · 开篇：什么是 Sowork AI 企业智能ERP系统

> 这是整个教程系列的第 0 篇，也是你阅读后续 46 篇的「地图」。不管你是企业管理者还是开发者，都建议先读这一篇——它回答三个问题：**这是什么、能帮你解决什么、你该按什么顺序学**。

## 学习目标

- 用一句话说清 Sowork AI 企业智能ERP系统的定位与边界。
- 看懂它的「能力全景图」：HR、ERP、CMS、商城、AI、低代码、系统管控分别管什么。
- 理解它为什么是「单体全栈」而不是一堆微服务，以及这对你的成本意味着什么。
- 拿到两条学习路径（管理者路径 / 开发者路径），知道每篇教程该怎么读。
- 记住 6 条一致性红线（产品名、版本、账号、端口、依赖安装方式、路由事实），避免后续踩坑。

## 核心概念

很多开源 ERP 给人的印象是「半成品」：要么只有财务，要么只有 HR，要么是个空壳低代码。Sowork AI 的不同之处在于——**它把一家企业日常运转需要的绝大部分系统，装进了一套代码里**。

```
                  Sowork AI 企业智能ERP系统 (v1.2.0, Apache 2.0)
   ┌──────────────────────────────────────────────────────────────────┐
   │  表现层 (React 19 + Ant Design 5)                                   │
   │   后台管理端  │  网站前台(/site)  │  商城前台(/shop)  │  AI 对话台    │
   ├──────────────────────────────────────────────────────────────────┤
   │  HR  │  ERP(产品/PLM/仓储/销售/采购/生产/财务/质量)                  │
   │  CMS │  商城 ShopXO 超集  │  AI(RAG/工具循环)  │  低代码(模型热部署) │
   ├──────────────────────────────────────────────────────────────────┤
   │  企业级管控：RBAC 权限点 + 站点级作用域 + i18n 多语言                 │
   ├──────────────────────────────────────────────────────────────────┤
   │  后端 (Node 22 + Express 4)  │  自研 Odoo 风格 ORM  │  数据层         │
   │  通用 CRUD /api/:table + 命名路由组  │  SQLite / PostgreSQL          │
   └──────────────────────────────────────────────────────────────────┘
```

**为什么是单体全栈？** 对中小企业来说，一套 `npm start` 就能跑起来的系统，比「启动 8 个微服务 + 1 个网关 + 1 个注册中心」成本低得多：少一个组件就少一个故障点，少一份运维文档。Sowork AI 选择「单体应用 + 单库优先（SQLite，可切 PG）」，正是为了**私有化部署的极简**——这也是它敢说「5 分钟跑起来」的底气（见第 02 篇）。

## 界面布局与操作流程

你启动系统后，用 `admin / admin123` 登录，会看到左侧一条**按权限动态渲染**的导航栏（源码 `client/src/components/Layout.tsx:46-308`）。它把所有功能分成几类：

- **顶部单页**：首页仪表盘 `/`、组织管理 `/organization`、数据统计 `/statistics`、低代码 `/lowcode`、应用管理 `/apps-manager`。
- **可折叠业务模块**：人事、考勤、薪酬、绩效、招聘、后勤、培训、审批、产品、PLM、仓储、销售、采购、生产、财务、质量、CMS、商城、AI、系统……每个模块展开后还有若干子页。
- **前台入口**：导航里「🌐 访问网站」跳 `/site`（内容门户），「🛒 访问商城」跳 `/shop`（在线商城），都不需要登录即可浏览。

> 管理者视角快速上手：登录后先看「首页仪表盘」，那里是经营数据的聚合视图；想找某个业务，就在左侧导航按模块点开即可，无需关心背后技术。

## 底层逻辑与数据模型

Sowork AI 的「模块」在后端有两套实现约定，这点开发者必须提前知道（避免后面看源码迷惑）：

1. **平台内核**（`server/modules/`）：ORM、数据库驱动、认证、登录安全等基础设施。
2. **业务模块**：
   - `modules/`（Odoo 风格，带 `manifest.json` + `models/models.js`，如 `hr_core`、`odoo_base`）；
   - `addons/`（低代码向导生成或插件，如 `app_mru06doi`、`analytic_odoo`）；
   - 数据访问统一走 `GET/POST/PUT/DELETE /api/:table`（表名在 `ALLOWED` 白名单内），少量复杂业务用命名路由组（如 `/attendance/clock-in`、`/salary/calculate`、`/ai/chat`）。

```
典型一次「看员工列表」的请求：
浏览器 → /personnel (React 路由)
       → useAPI.get('/api/employees')   ← 命中 catch-all /api/:table
       → apiAuthMiddleware 校验 token
       → ALLOWED 白名单校验 'employees' 通过
       → 查 SQLite/PG，返回 JSON → 表格渲染
```

## 源码剖析

入口文件是 `server/standalone.ts`。启动顺序（约 `standalone.ts:61-155`）：

```ts
// server/standalone.ts (节选，行号约为示意区间 61-155)
const db = new DatabaseService();        // 1. 初始化数据库（WAL 模式）
await db.onModuleInit();
const env = new Environment(db);          // 2. 建 Environment（ORM 上下文）
await ModelRegistry.loadFromModule(...);  // 3. 扫描并注册所有模型 → 自动建表
await seedCmsShop.default(db);            // 4. 播种种子数据（地区/配置等）
// 5. express.json + loginSecurityMiddleware → apiRouter() → 静态资源 → listen
```

关键点：**模型的注册即建表**。`ModelRegistry.loadFromModule` 会清掉 require 缓存后加载每个模块的 `models.js`，发现新表就 `CREATE TABLE`，发现新字段就 `ALTER TABLE ADD COLUMN`——你加一个模块，数据库结构自动跟上，无需手写迁移脚本。这正是「低代码热部署」（第 11 篇）能成立的前提。

前端入口 `client/src/index.tsx` 挂载 `BrowserRouter` + 两个 Provider；`app.tsx:173-390` 注册了全部路由；`Layout.tsx` 根据角色过滤可见模块（超级管理员 `/system` 才可见）。

## 原理剖析

> **单体 ≠ 简陋**。Sowork AI 用「集中式 `apiRouter()` + 通用 CRUD + 白名单」替代了「每个模块一个子路由文件」的散养模式。好处是：新增一张表几乎零路由代码；坏处是权限必须靠 `ALLOWED` 白名单 + `apiAuthMiddleware` 兜底，不能指望「没写路由就访问不到」。这也是为什么第 08 篇要把 RBAC 讲透——**前端隐藏按钮 ≠ 后端拦得住**，安全靠的是 `resolve` 返回的权限点 + 中间件校验。

## 管理者视角

| 维度 | 价值 | 风险 / 注意 |
|------|------|------------|
| 一体化 | HR/ERP/CMS/商城/AI 一套系统、一个账号、一份数据，告别「系统孤岛」 | 单库起步，数据量大了要规划 PG 迁移（第 04 篇） |
| 私有化 | 可内网部署，数据不出公司；SQLite 零运维，PG 可上云 | 务必做好备份（第 45 篇） |
| 成本 | 开源 Apache 2.0，无授权费；单体部署省服务器 | 需要 1 名会 Node 的同事做基础运维 |
| AI 内置 | 知识库问答、BI、智能预警开箱即用，不用另买 SaaS | AI 能力依赖你接入的模型（OpenAI / 本地 Ollama） |
| 可扩展 | 低代码 4 步生成模块，非程序员也能搭应用 | 低代码生成的是标准 CRUD，复杂逻辑仍需开发 |

## 注意事项

- ⚠️ **产品名是 Sowork AI 企业智能ERP系统**，系列所有文章、标题、海报统一用这个，不要混用旧名。

- ⚠️ 默认账号 `admin/admin123` **上线必须改密码**（第 40 篇系统管理会讲）。
- 🔑 端口默认 3000，可用环境变量 `SERVER_PORT` 覆盖。
