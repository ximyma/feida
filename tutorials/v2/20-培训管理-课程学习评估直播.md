## 核心概念

培训模块的四个子功能，职责各不相同：

| 子功能 | 一句话职责 | 核心表 |
| --- | --- | --- |
| 课程（含章节/视频） | 把知识封装成可分发的内容 | `training_courses_v2` / `training_chapters` |
| 学习 | 员工消费课程内容，系统记录进度 | `training_learning_progress` |
| 评估 | 学完即考，收集反馈与评分 | `training_reviews` / `assessment_templates` |
| 直播 | 实时授课、互动与签到 | `training_live_sessions` |

🔑 要记住：培训数据全部走 catch-all 路由 `/api/:table`，**不存在** `/api/hr` 这种按域前缀划分的接口。你在前端看到的 `API.get('training_courses')` 实际就是 `GET /api/training_courses`。

## 界面布局与操作流程

侧边栏在 `client/src/components/Layout.tsx:112` 仅为培训注册了两个入口：

```
/training        培训首页
/training/course 课程管理
```

但真正的丰富界面在 `TrainingPage.tsx`，它用一个 `Tabs` 渲染了 12 个页签（`TrainingPage.tsx:672`）：

```
课程中心 ─ 我的学习 ─ 直播中心 ─ 培训计划 ─ 在线课程(旧)
培训班 ─ 培训记录 ─ 学习进度 ─ 培训推送 ─ 评估管理 ─ 学习报表 ─ 我的证书
```

典型操作流程如下：

1. **建计划**：在「培训计划」页签填写目标部门、负责人、预算与周期。
2. **建课程**：在「课程中心(新)」创建 `training_courses_v2`，再为其添加章节 `training_chapters`。
3. **发课程**：章节齐全后「发布」，状态由 `draft` 变为 `published`。
4. **员工学**：员工在「我的学习」观看视频，系统通过 `/training/v2/progress` 保存断点。
5. **做评估**：学完在课程详情写评价/评分，数据入 `training_reviews`。
6. **开直播**：在「直播中心」创建并 `start` 一场 `training_live_sessions`。

## 底层逻辑与数据模型

培训域在 `server/standalone.ts:1081` 被整体注册进 ALLOWED 白名单：

```ts
'training_plans','training_courses','training_records','training_evaluations',
'training_learning_progress','training_notifications',
'training_categories','training_courses_v2','training_chapters',
'training_reviews','training_review_replies','training_notes',
'training_live_sessions','training_live_messages','training_live_attendances',
'training_live_reservations','training_learning_paths','training_path_courses',
'training_classes','assessment_templates',
```

关键关系用 ASCII 树表示：

```
training_courses_v2 (1)
   └─ training_chapters (N)        章节/视频/考试
        └─ training_learning_progress (员工×课程)
             ├─ progressPercent
             ├─ totalWatchTime
             └─ watchCount
training_reviews (N)  ── 评价/评分，挂在 courseId 下
training_live_sessions (N) ── 直播，关联 chapterId
```

课程发布时，服务端把状态写为 `published` 并打 `publishedAt` 时间戳（`standalone.ts:5095`）。只有 `published` 的课程会出现在课程中心列表——列表接口默认追加 `AND status = 'published'`（`standalone.ts:4915`）。

## 源码剖析

### 1. 视频上传走独立端点

课程视频不走 `/api/:table`，而是专用接口 `POST /training/video/upload`（`standalone.ts:2407`）。它用 `multer` 接收文件，校验扩展名白名单 `.mp4/.webm/.ogg/.mov/.avi/.mkv`，再用 `ffprobe` 抽取时长，最后返回可播放的 `/api/training/video/:videoId`。

```ts
router.post('/training/video/upload', upload.single('file'), async (req, res) => {
  const { chapterId, courseId, uploaderId } = req.body;
  // ...校验类型、重命名、ffprobe 取时长
  const videoUrl = `/api/training/video/${videoId}`;   // standalone.ts:2447
});
```

### 2. 章节学习进度与防刷

员工每看一段，前端调用 `POST /training/v2/progress`（`standalone.ts:5222`）。服务端计算进度百分比，并**只增不减**地更新：

```ts
progressPercent = Math.max(existing.progressPercent || 0, progressPercent), // standalone.ts:5269
```

这就是「学习进度防刷」的核心：无论你回拖进度条多少次，进度只会取历史最大值，无法靠反复拉满来作弊刷完成。视频观看时长也按 `Math.abs(position - lastPosition)` 增量累加（`standalone.ts:5246`）。

完成判定：`progressPercent >= 100` 即 `completed`，同时回写课程的 `completionCount`（`standalone.ts:5305`）。

### 3. 断点续看

`POST /training/progress/video-position`（`standalone.ts:4667`）记录播放位置。下次进入时 `GET /training/progress/video/:courseId`（`standalone.ts:4719`）返回上次 `videoPosition`，实现「从上次停止处继续」。

### 4. 直播与签到

创建直播 `POST /training/live`（`standalone.ts:5577`）生成 `streamKey`、推流/拉流地址，初始 `status='pending'`；点击开始走 `PUT /training/live/:id/start` 置为 `live`（`standalone.ts:5619`）。学员进入会产生 `training_live_attendances` 签到记录。

## 原理剖析

为什么进度能「防刷」又能「断点」？本质是**服务端为「员工×课程」维护唯一一行** `training_learning_progress`（`standalone.ts:5238` 的 `SELECT ... WHERE employeeId = ? AND courseId = ?`）。命中则更新，未命中则插入。所有进度计算都发生在服务端，前端只是把播放器时间上报，无法自行写库造假。

评估反馈同样服务端聚合：`training_reviews` 写入后，服务端重算课程的 `rating` 与 `reviewCount`（`standalone.ts:5444`），课程详情接口再把评价统计一并带出（`standalone.ts:4958`）。

## 管理者视角

把培训当资产，而不是一次性活动：

- **资产沉淀**：每门课程、每个章节、每条评价都落库，离职也不会丢失，新人可直接复用。
- **合规持证上岗**：`training_records` 与 `training_certificates` 记录成绩与证书，配合「必修」标记（`isMandatory`），可一键查出「未持证」员工（`standalone.ts:5485` 的未完成必修查询）。
- **与绩效/人才发展联动**：培训完成率、平均分已在「学习报表」汇总，可作为晋升与绩效面谈的硬数据。

## 注意事项

⚠️ 视频必须通过 `POST /training/video/upload` 上传，直接把 URL 写进课程字段无法走服务端转码与防盗链。

🔑 学习进度由服务端取历史最大值，前端无法刷进度；若需重置，要在数据库层面清 `training_learning_progress` 行。

✅ 课程评价 `training_reviews` 支持匿名（`status='published'` 后对外展示），员工可放心给出真实反馈。

⚠️ 直播地址当前为占位 `rtmp://your-stream-server.com/...`，接入真实推流需替换 `streamUrl`/`pullUrl` 配置。
