## 三、架构对比

### 3.1 数据库

| 维度 | 飞达 | Odoo |
|------|------|------|
| 数据库 | SQLite (better-sqlite3) | PostgreSQL 13+ |
| 并发能力 | 单写入锁 (WAL 模式缓解) | 完整 MVCC 并发 |
| 高可用 | ❌ 单文件 | ✅ 主从复制/集群 |

## 四、功能差距汇总

### 🔴 P0 - 必须补齐（企业级门槛）

| 差距 | 影响 | Odoo 参照 |
|------|------|-----------|
| **SQLite → PostgreSQL** | 无法支撑 50+ 并发写入，不能做主从备份 | `psycopg2 + pg_dump` |
| **行级权限** | 无法实现"部门经理只看本部门" | `ir.rule + domain` |
| **API 认证** | 所有 `/api/*` 无鉴权，安全隐患大 | `API Key / OAuth2` |
| **登录安全** | 无 2FA、无登录锁定 | `auth_totp / auth_ldap` |

### 🟡 P1 - 重要增强（竞争力提升）

| 差距 | Odoo 参照 | 实现难度 |
|------|-----------|----------|
| **模块化系统** | `__manifest__.py + 一键安装` | 高 |
| **记录级权限** | `ir.rule` 行级过滤 | 中 |
| **工作流引擎增强** | 状态机 + 自动化动作 + 定时任务 | 中 |
| **完整会计系统** | 科目表/记账/对账/报表/税务 | 高 |
| **MRP 运算引擎** | 物料需求计划/生产排程 | 高 |
| **AI 记忆系统** | 对话历史持久化 + 自动摘要 | 低 |
| **AI 技能系统** | 可插拔技能注册/发现/管理 | 中 |
| **多公司/多组织** | `company_id` 多租户 | 中 |

### 🟢 P2 - 锦上添花（差异化优势）

| 差距 | Odoo 参照 | 实现难度 |
|------|-----------|----------|
| **博客模块** | `website_blog` | 低 |

## 五、升级路线建议

### 第一阶段：企业级基础（1-2 周）

```
PostgreSQL 迁移 → API 认证 → 登录安全 → 行级权限
```

将 SQLite 改为 PostgreSQL（`pg` 包已在 dependencies），实现：
- 数据库连接池
- `pg_dump` 定时备份
- JWT API Token 认证
- 登录失败锁定 + 2FA (可选)
- 行级数据隔离（部门/员工级）

### 第二阶段：AI 增强（1 周）

```
对话持久化 → 记忆系统 → 技能管理器
```

参照 CowAgent 架构：
- 对话历史 SQLite → PostgreSQL 持久化
- 自动摘要 + 嵌入缓存
- 可插拔技能注册系统

### 第三阶段：核心模块深化（2-4 周）

```
工作流引擎增强 → 完整审批 → 财务管理 → 中国本土化
```

参照 Odoo 模块：
- 状态机 + 自动化动作触发器
- 中国会计科目表集成
- 支付宝/微信支付接入
- 鞋服行业款色码（`shoe_garment` 已有完整方案文档）

### 第四阶段：生态建设（长期）

```
模块市场 → 插件 SDK → Admin CLI
```

## 六、关键结论

1. **飞达在特定领域有优势**：AI 代码助手/工具执行、电商营销玩法（秒杀/拼团/分销）、CMS 标签/SEO、BI 看板。这些是 Odoo 社区版不提供的。

2. **Odoo 在企业级基础上远超飞达**：PostgreSQL 高并发、行级权限、完整会计/MRP、模块生态、多公司支持。飞达要支撑真实企业使用，这些必须补上。

3. **CowAgent (AI Agent) 架构值得深度借鉴**：486 文件的 Agent 系统中，"记忆/技能/进化"三层架构可直接移植到飞达。

4. **飞达的技术栈（React+TypeScript）比 Odoo 的 QWeb/OWL 更现代**：前端开发体验、npm 生态、AI 集成便利性都是飞达的优势。

5. **SQLite → PostgreSQL 是一切的前提**：不做这个迁移，后续所有企业级特性都缺乏根基。
