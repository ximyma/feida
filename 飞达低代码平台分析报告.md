## 二、核心缺陷

### 2.1 致命问题 ❌

| # | 问题 | 影响 |
|----|------|------|
| 1 | **`glob` 路径解析失败** — Agent 用 `glob({"pattern":"client/src/pages/AppsPage/**/*.tsx"})` 找不到实际文件，因为工作目录是 `/d/feida` 但工具不支持相对路径前缀 (`client/`) | 所有通过 Agent 的源码分析无法工作 |
| 2 | **`many2one` 只显示 ID** — 关联数据不解析，用户看到 `city_id: "c001"` 而非 "北京" | 表单/列表不可用 |
| 3 | **`one2many` 无子表编辑** — 只能看到关联模型名，无法在父表单内编辑子记录 | 一对多关系不可用 |
| 4 | **无行级/字段级权限** — 低代码生成的应用所有人可读写 | 无法用于生产 |
| 5 | **工作流未集成** — `automation-engine.ts` 和 `workflow-engine.ts` 与低代码应用完全独立 | 无法做审批/状态流转 |

### 2.2 严重不足 ❗

| # | 问题 | 建议 |
|----|------|------|
| 6 | 无货币/邮箱/URL/颜色/富文本字段类型 | 字段画廊补充12+类型 |
| 7 | 搜索只有全局文本，无高级筛选/字段级过滤/保存筛选 | 增加 FilterPanel |
| 8 | 无 Excel/CSV 导入导出 | 数据处理必需 |
| 9 | 无看板(Kanban)视图 | 拖拽式状态管理不可用 |
| 10 | 无图表/日历/甘特图视图 | 可视化不足 |
| 11 | 表单布局只有垂直排列，无分组/Tab/面板/网格 | 复杂表单难用 |

### 2.3 一般不足 📋

| # | 问题 | 建议 |
|----|------|------|
| 12 | 字段验证仅必填，无正则/范围/唯一性 | 增强验证引擎 |
| 13 | 公式引擎仅四则运算，无聚合/日期函数 | 扩展公式语法 |

## 三、改进方案

### Phase 1: 关键修复 (1-2 周) 🔴

**1.1 修复路径解析**
```
问题: Agent 用 glob("client/src/...") 找不到文件
修复: 在 bash/grep/glob 工具执行前自动注入 workdir (process.cwd())
      或在工具注册时设置 workingDirectory
```

**1.2 many2one 下拉搜索**
```
问题: 关联字段只显示 ID
修复: FormView 中 many2one 用 Select(showSearch) + api 分页搜索
      后端增加 /api/model/:table/search?q=关键字 模糊搜索
```

**1.3 Excel/CSV 导入导出**
```
导出: GET /api/model/:table/export?format=csv → 下载
导入: POST /api/model/:table/import → 批量创建 (含错误行提示)
前端: Upload.Dragger + 预览表格 + 确认导入
```

### Phase 2: 视图增强 (2-3 周) 🟡

**2.1 看板视图 (KanbanView)**
```
按状态字段分组 → 列
卡片拖拽改变状态 → 自动写入 DB
参照: Odoo kanban view
新增: POST /api/model/:table/browse/:id/write (拖拽后更新)
```

**2.2 图表视图 (GraphView)**
```
引入: ECharts (体积小)
选择: X轴字段、Y轴字段、聚合函数
类型: 柱状图/折线图/饼图
后端: GET /api/model/:table/aggregate?groupBy=X&aggregate=Y:sum
```

**2.3 高级筛选 (FilterPanel)**
```
字段级过滤: 等于/包含/大于/小于/范围内/为空
AND/OR 组合: 多条件逻辑
保存筛选: 命名+持久化到 localStorage
```

### Phase 3: 平台化 (3-4 周) 🟢

**3.1 权限集成**
```
安全配置文件自动生成: security/access.json
模型级: 创建/读取/写入/删除 四权限
行级: ir.rule (按 company_id/department 过滤)
字段级: 敏感字段隐藏/只读
前端: usePermission + can() 控制按钮显隐
```

**3.2 工作流集成**
```
模块声明: app.json → "workflow": true
自动注册: 每个表绑定状态机
操作: 提交/审批/驳回/撤回 → 状态流转
UI: 审批按钮 + 流程进度条
```

**3.3 布局引擎**
```typescript
interface FormLayout {
  type: 'group' | 'tab' | 'grid';
  label?: string;
  columns?: number;
  children: Array<FormLayout | string>;
}
// TemplateDesigner 中拖拽字段到布局容器
// FormView 中解析渲染
```

### Phase 4: 完善 (持续) 🔵

- 补充字段类型: 货币/邮箱/URL/颜色/富文本(Tiptap)/附件
- 公式引擎扩展: SUM/AVG/COUNT/聚合 + 日期函数
- 字段验证: 正则/范围/唯一/自定义 JS 表达式
- 国际化: `_t()` + 翻译表 + language switcher
- 虚拟滚动: react-window 优化大数据量列表
- API 文档: 低代码生成 OpenAPI spec
