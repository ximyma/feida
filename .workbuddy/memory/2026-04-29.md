## 完成功能：工作流设计器保存功能修复

### 问题描述
流程审批模块的"工作流设计"Tab无法保存新建或修改的工作流。前端调用了 `POST /api/workflows` 和 `PUT /api/workflows/:id`，但后端缺少这两个 API。

### 修复的文件
**`server/standalone.ts`** - 添加缺失的 API 路由：

1. **POST /api/workflows** - 创建工作流定义
   - 参数：name, code, description, nodes, edges, status, isDefault
   - 自动生成 ID 和时间戳
   - 处理前端已序列化的 nodes/edges 数据

2. **PUT /api/workflows/:id** - 更新工作流定义
   - 参数：同创建
   - 自动递增版本号
   - 检查工作流是否存在

3. **DELETE /api/workflows/:id** - 删除工作流
   - 检查是否有正在运行的实例
   - 有实例时不允许删除

### 修复的数据序列化问题
前端在发送前已将 `nodes` 和 `edges` JSON序列化，后端需要检测：
- 如果传入的是字符串，则不再序列化
- 如果传入的是数组，则序列化

4. 测试修改已存在的工作流
5. 测试添加节点、连线、删除节点等操作
6. 点击"保存"验证修改是否持久化
